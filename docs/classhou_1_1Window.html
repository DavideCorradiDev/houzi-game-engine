hou::window::window ( const std::string &  title,
const video_mode &  vm,
window_style  style 
)

Creates a window object.

The window client size corresponds to the resolution in vm. The window client is be positioned in the middle of the screen. The window uses the default system icon. The window is not visible. The window does not grab the mouse cursor. Key repeat is disabled for the window.

Parameters
titlethe title of the window.
vmthe video mode of the window, specifying its client size and bytes per pixel.
stylethe style of the window. Only one window can be fullscreen. If fullscreen mode is specified and vm is not a valid fullscreen video_mode, an error will be thrown.
Exceptions
hou::precond_errorif style is fullscreen and a fullscreen window already exists.
hou::os_errorif the window could not be created.

Pops the front event in the queue of this window.

This function does not automatically update the queue. Call update_event_queue before to fill it with events coming from the OS. This function is not blocking. If the event queue is empty, an event of type empty is returned.

◆ wait_event()

window_event hou::window::wait_event ( )

Waits for the event queue of this window to contain at least one event and pops it.

This function always updates the event queue, so a call to update_event_queue is not necessary. This function is blocking, it will keep updating and checking the queue until an event is pushed. This function only pops the first event. It might be that multiple events have been inserted in the queue and therefore the queue might still be filled.

Exceptions
hou::os_errorin case of error getting the window events.
std::bad_alloc.
Returns
the front event.
